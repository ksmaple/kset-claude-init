# 规则名称
Git 提交规范

# 规则摘要
中文提交信息，携带类型前缀，标题不超过 50 字。

# 规则具体内容

## 基本格式
- 使用中文撰写提交信息
- 必须携带类型前缀，格式为 `<类型>: <标题>`

## 常用类型前缀
| 前缀 | 用途 |
|------|------|
| `feat:` | 新功能 |
| `fix:` | 修复 bug |
| `docs:` | 文档更新 |
| `style:` | 代码格式调整（不影响逻辑） |
| `refactor:` | 重构 |
| `test:` | 测试相关 |
| `chore:` | 构建/工具/依赖更新 |
| `perf:` | 性能优化 |

## 标题要求
- 简洁明了，不超过 50 字
- 使用祈使语气，避免无意义词汇
- 说明做了什么，而非罗列文件

## 正文与 Footer
- 需要时在正文说明**为什么改**、**影响范围**、**注意事项**
- 有关联 issue / PR 时，在 footer 标注：`关联 #123`
