# 规则名称
代码风格规范

# 规则摘要
命名、格式、注释与修改纪律，拒绝过早抽象。

# 规则具体内容

## 通用原则
- 保持简洁可读
- 遵循项目已有风格
- 不为了风格一致而重构无关代码
- 拒绝过早抽象：三个相似代码块优于一个假设性的通用工具函数

## 命名规范
- 变量/函数: 有意义的命名，避免单字母（循环变量除外）
- 常量: 全大写蛇形命名 `MAX_COUNT`
- 类/组件: PascalCase
- 文件/目录: kebab-case

## 代码格式
- 使用项目配置的格式化工具
- 提交前通过 lint 检查

## 注释规范
- 优先自解释代码
- 仅在复杂逻辑、边界情况处添加注释
- 注释解释"为什么"，而非"做了什么"

## 修改纪律
- 一次改动只解决一个问题
- 修复 bug 不顺便重构无关代码
- 新增功能不改变未涉及模块的格式
