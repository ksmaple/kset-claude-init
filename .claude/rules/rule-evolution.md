# 规则名称
规则自主优化

# 规则摘要
当某个未被文档化的约束或行为模式在任务执行过程中被反复使用时，自动将其提炼为 `.claude.local/rules/` 下的本地规则文档，并向用户说明，持续完善配置体系。

# 规则具体内容

## 触发条件
同时满足以下两点时，触发规则自动生成：
1. **未被文档化**：该约束、习惯或行为模式尚未写入 `.claude/rules/` 或 `.claude.local/rules/` 的正式规则文件中；
2. **反复出现**：在本次对话中被引用、执行或强调达到 **3 次及以上**，或在多个独立任务中持续出现。

## 自动生成流程
1. 识别并总结反复出现的约束或行为模式
2. 按三段式结构撰写规则内容（规则名称、规则摘要、规则具体内容）
3. 将规则文件写入 `.claude.local/rules/<kebab-case名称>.md`
4. 更新 `CLAUDE.local.md` 的本地配置索引，追加新规则路径
5. 向用户简要说明已生成的规则文件、核心内容及存放位置

## 存放位置
- **一律自动放入** `.claude.local/rules/`（个人本地配置）
- 若用户后续明确要求提升为团队通用约定，再迁移至 `.claude/rules/`（公共配置）

## 文档化要求
新提炼的规则必须采用三段式结构：
1. **规则名称**
2. **规则摘要**
3. **规则具体内容**
