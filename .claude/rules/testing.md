# 测试规范

## 基本要求
- 新功能必须包含对应的测试
- Bug 修复必须包含回归测试
- 测试应当聚焦于行为，而非实现细节

## 测试类型
- **单元测试**: 覆盖核心逻辑和边界条件
- **集成测试**: 覆盖模块间交互
- **E2E 测试**: 覆盖关键用户路径（如适用）

## 测试原则
- 一个测试只验证一个概念
- 测试名称应清晰描述被验证的行为
- 避免过度使用 mock，优先测试真实行为
- 不要为了 100% 覆盖率而写无意义的测试

## 修改纪律
- 运行与修改相关的测试，确保通过后再提交
- 如果项目没有测试基础设施，不要花费大量时间搭建
- 不因为"以后可能需要"而为简单代码添加复杂的测试框架
