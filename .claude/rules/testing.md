# 规则名称
测试规范

# 规则摘要
新功能/Bug 修复需附测试，聚焦行为而非实现细节。

# 规则具体内容

## 基本要求
- 新功能必须包含对应测试
- Bug 修复必须包含回归测试
- 测试聚焦行为，而非实现细节

## 测试类型
- 单元测试：覆盖核心逻辑和边界条件
- 集成测试：覆盖模块间交互
- E2E 测试：覆盖关键用户路径（如适用）

## 测试原则
- 一个测试只验证一个概念
- 测试名称清晰描述被验证的行为
- 避免过度 mock，优先测试真实行为
- 不为 100% 覆盖率写无意义测试

## 修改纪律
- 运行相关测试，确保通过后再提交
- 无测试基础设施的项目，不花时间搭建
- 不为简单代码添加复杂测试框架
