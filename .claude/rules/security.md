# 规则名称
安全要求

# 规则摘要
代码安全、依赖安全、密钥管理与泄露应急。

# 规则具体内容

## 代码安全
- 不信任用户输入，做验证过滤
- 避免命令注入、SQL 注入、XSS
- 不硬编码密钥、Token
- 不猜测 URL

## 依赖安全
- 评估新依赖的维护状态和安全记录
- 优先官方或广泛认可的库
- 及时更新有漏洞的依赖

## 数据处理
- 最小权限原则
- 敏感数据加密脱敏
- 遵守隐私法规

## 密钥与配置
- 配置文件（.env、config.json 等）不硬编码敏感信息
- 使用环境变量或密钥管理服务
- 敏感配置路径加入 .gitignore
- 提供 .env.example 等模板（值为空）

## 文档与共享
- 不在文档、Issue、PR、评论中写入密钥
- 截图时打码密钥、Token、Cookie
- 明文密钥不通过 IM 发送

## 泄露应急
- 立即轮换已泄露的密钥
- 清理 Git 历史（git filter-repo / BFG Repo-Cleaner）
- CI 中接入密钥扫描工具（gitleaks、truffleHog）
