# git-commit

## 描述

根据当前暂存区变更，生成符合团队 Git 提交规范的提交信息，或对已有提交信息进行优化。

## 使用方式

```bash
/git-commit
```

或提供已有提交信息进行优化：

```bash
/git-commit "<原始提交信息>"
```

## 执行流程

1. 读取 `.claude/rules/git-commit.md` 了解团队提交规范
2. 分析 `git diff --staged` 中的变更内容
3. 判断最合适的类型前缀和标题
4. 生成符合规范的提交信息（标题 + 可选正文 + 可选 footer）
5. 说明推荐理由

## 约束

- 严格遵循 `<类型>: <中文标题>` 格式
- 标题控制在 50 字以内
- 不自动执行 `git commit`，仅生成文案供用户确认
